# توثيق مشروع نظام التنبؤ بأمراض الكلى (Kidney Disease Prediction System)

##  نبذة عن المشروع
هذا المشروع عبارة عن نظام ذكي متكامل يهدف إلى مساعدة الأطباء والمرضى في **الكشف المبكر عن مرض الكلى المزمن (CKD)** وتحديد مرحلته بدقة. يعتمد النظام على تقنيات الذكاء الاصطناعي المتقدمة لتحليل البيانات الطبية وصور التحاليل، مع توفير مساعد ذكي للإجابة على الاستفسارات الطبية.

---

##  المميزات الرئيسية (Key Features)

1.  **التنبؤ بالمرض (Prediction):** استخدام خوارزميات تعلم الآلة (ML) والتعلم العميق (DL) لتحديد ما إذا كان المريض مصاباً بمرض الكلى أم لا.
2.  **تحليل صور التحاليل (OCR):** استخراج البيانات تلقائياً من صور التقارير الطبية بدلاً من إدخالها يدوياً.
3.  **تحليل الأشعة المقطعية (CT Scan):** استخدام نماذج التعلم العميق (MobileNetV2) لاكتشاف الأورام والحصوات في أشعة الكلى.
4.  **تحديد المرحلة (Staging):** حساب معدل الترشيح الكبيبي (eGFR) وتحديد مرحلة المرض (G1-G5) ومستوى الخطورة بناءً على بروتوكولات KDIGO العالمية.
5.  **المساعد الذكي (RAG Chatbot):** شات بوت طبي يعتمد على Google Gemini 1.5 Flash للإجابة على أسئلة المرضى بناءً على قاعدة معرفة طبية موثوقة.
6.  **النظام الغذائي الذكي (Smart Diet Planner):** توليد خطة غذائية لـ 7 أيام تناسب حالة المريض ونسب المعادن لديه بناءً على توصيات KDIGO.
7.  **التقارير الطبية (Reports):** إنشاء تقارير PDF احترافية تحتوي على نتائج التشخيص والتوصيات.

---

## ️ التقنيات والمكتبات المستخدمة (Tech Stack)

تم بناء المشروع باستخدام لغة **Python 3.10+**، وتم استخدام المكتبات التالية:

### 1. الذكاء الاصطناعي وتعلم الآلة (AI & ML)
*   **Scikit-learn:** لبناء نماذج التعلم التقليدية (SVM, Random Forest).
*   **XGBoost:** لبناء نموذج Gradient Boosting عالي الكفاءة.
*   **TensorFlow / Keras:** لبناء الشبكات العصبية (Neural Networks) للتعلم العميق.
*   **Imbalanced-learn:** لمعالجة عدم توازن البيانات (SMOTE).

### 2. معالجة الصور والنصوص (OCR)
*   **EasyOCR:** المكتبة الأساسية لاستخراج النصوص والأرقام من صور التحاليل (تدعم العربية والإنجليزية).
*   **Tesseract:** محرك OCR بديل لزيادة الدقة.
*   **OpenCV & Pillow:** لمعالجة الصور وتحسين جودتها قبل استخراج النصوص.

### 3. الذكاء الاصطناعي التوليدي (Generative AI & RAG)
*   **Google Gemini API (gemini-1.5-flash):** النموذج اللغوي المستخدم في الشات بوت للإجابة والتحليل.
*   **ChromaDB:** قاعدة بيانات المتجهات (Vector Database) لتخزين واسترجاع المعلومات الطبية (RAG).

### 4. معالجة البيانات (Data Processing)
*   **Pandas:** للتعامل مع ملفات البيانات (CSV, Excel) وتحليل الجداول.
*   **NumPy:** للعمليات الحسابية والمصفوفات.

### 5. الواجهة الخلفية والتقارير (Backend & Reports)
*   **FastAPI:** (اختياري) لبناء واجهة برمجة تطبيقات (API) سريعة لربط النظام بتطبيقات الويب أو الموبايل.
*   **ReportLab:** لإنشاء ملفات PDF ديناميكية تحتوي على التقارير الطبية.

## ️ هيكلية النظام (System Architecture)

يعمل النظام من خلال عدة وحدات مترابطة (Modules):

1.  **`src/preprocessing`**: مسؤول عن تنظيف البيانات، معالجة القيم المفقودة، وتجهيزها للموديلات.
2.  **`src/models`**: يحتوي على كود تدريب الموديلات (Ensemble Model) الذي يدمج بين ML و DL لأفضل دقة.
3.  **`src/ocr`**: يستقبل صورة التحليل ⬅️ يعالج الصورة ⬅️ يستخرج الأرقام (Creatinine, Age, etc.).
4.  **`src/imaging`**: يستقبل صورة الأشعة المقطعية ⬅️ يمررها على MobileNetV2 ⬅️ يكتشف الفئة (طبيعي، كيس، حصوة، ورم).
5.  **`src/staging`**: يأخذ النتائج ⬅️ يحسب eGFR ⬅️ يحدد مرحلة المرض والخطورة.
6.  **`src/rag`**: نظام البحث الدلالي للإجابة على أسئلة المستخدم (RAG).
7.  **`src/diet`**: يأخذ القيم الطبية ويولد جدول غذائي ذكي.

##  طريقة التشغيل (How to Run)

### 1. تثبيت المكتبات
```bash
pip install -r requirements.txt
```

### 2. تدريب النظام (Training)
لإعادة تدريب الموديلات على البيانات الجديدة:
```bash
python main.py train
```
*سيتم حفظ الموديلات المدربة في مجلد `models/`.*

### 3. عمل تنبؤ (Prediction)
للتنبؤ بحالة مريض باستخدام بيانات نصية:
```bash
python main.py predict --features "age=50,creatinine=1.2"
```

أو عن طريق صورة تحليل:
```bash
python main.py predict --image "path/to/report.jpg"
```
