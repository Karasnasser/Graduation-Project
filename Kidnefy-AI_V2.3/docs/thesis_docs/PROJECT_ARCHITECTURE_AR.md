# هيكلة وتصميم نظام التنبؤ بأمراض الكلى (Project Architecture)

##  نبذة عامة
هذا المشروع عبارة عن نظام ذكاء اصطناعي متكامل (End-to-End AI System) يهدف إلى:
1.  **التنبؤ المبكر** بمرض الكلى المزمن (CKD) واعتلال الكلى السكري.
2.  **تحليل الصور الطبية (OCR)** لاستخراج نتائج التحاليل تلقائياً.
3.  **تحليل الأشعة المقطعية (CT Scans)** للكشف عن الأورام والحصوات بالذكاء الاصطناعي.
4.  **تحديد مرحلة المرض (Staging)** بدقة باستخدام الذكاء الاصطناعي والمعادلات الطبية.
5.  **الإجابة الذكية (RAG Chatbot)** على استفسارات المرضى والأطباء باستخدام Google Gemini.
6.  **النظام الغذائي الذكي** لتوليد جدول طعام مخصص لمرضى الكلى.

## ️ الهيكلة البرمجية للمشروع (Project Structure)

تم تصميم المشروع بنظام الـ **Modular Architecture**، حيث كل وظيفة مستقلة في مجلد خاص بها داخل `src/` لسهولة الصيانة والتطوير.

├── requirements.txt        # (Dependencies) المكتبات المطلوبة للتشغيل
├── .gitignore              # (VersionControl) الملفات المستثناة من الرفع
│
├── src/                    # (Core) الكود الأساسي للمشروع
│   ├── models/             #  نماذج الذكاء الاصطناعي
│   │   ├── ml_models.py    # نماذج التعلم الآلي التقليدية (RF, XGB, SVM)
│   │   ├── dl_models.py    # نماذج التعلم العميق (Neural Networks)
│   │   └── staging_model.py # (جديد) نموذج تحديد المرحلة (Stage 0-5)
│   │
│   ├── preprocessing/      #  معالجة البيانات
│   │   ├── data_loader.py  # تحميل وتنظيف البيانات الأساسية
│   │   └── staging_data_loader.py # (جديد) تحميل بيانات المراحل
│   │
│   ├── ocr/                # ️ التعرف الضوئي على الحروف
│   │   └── extraction.py   # استخراج الأرقام من صور التحاليل (EasyOCR)
│   │
│   ├── staging/            #  الحسابات الطبية
│   │   └── gfr_calculator.py # حساب eGFR وتحديد الخطورة (KDIGO)
│   │
│   ├── rag/                #  الشات بوت الطبي
│   │   └── engine.py       # الربط مع Google Gemini وقاعدة المعرفة (ChromaDB)
│   │
│   └── reports/            #  التقارير

│   └── diabetes_prediction_dataset.csv    # بيانات السكري
│
└── generated_reports/      #  مخرجات التقارير النهائية

## ⚙️ المكونات التقنية بالتفصيل (Components Detail)

### 1. محرك التنبؤ الرئيسي (Prediction Engine)
- **الهدف**: معرفة هل الشخص مريض أم لا (Binary Classification).
- **التقنية**: Ensemble Learning (تصويت مرجح).
- **الخوارزميات**:
  - `Random Forest`: لقوة التعامل مع البيانات غير الخطية.
  - `XGBoost`: لسرعة ودقة التنبؤ (Gradient Boosting).
  - `SVM`: للفصل بين البيانات المعقدة.
  - `Deep Neural Network`: طبقات Dense لاستخراج الأنماط الخفية.
- **الدقة**: تصل إلى **98-99%**.

### 2. نظام تحديد المرحلة (AI Staging System) - **ميزة جديدة**
- **الهدف**: تحديد مرحلة المرض بدقة (من 0 إلى 5).
- **التقنية**: Multi-class Classification باستخدام XGBoost.
- **البيانات**: مدرب على 4000 سجل طبي حقيقي.
- **المخرجات**:
  - المرحلة المتوقعة (مثلاً: Stage 3b - G3b).
  - نسبة الثقة (Confidence Score).
  - احتمالية كل مرحلة (Class Probabilities).

### 3. الحسابات الطبية (Clinical Calculation)
- **الهدف**: تدقيق النتائج بمعادلات طبية معتمدة عالمياً.
- **المعادلة**: CKD-EPI 2021 للكبار (الأدق حالياً).
- **المعايير**: معايير KDIGO لتصنيف الخطورة (Low, Moderate, High, Very High).
- **الفائدة**: تعمل جنباً إلى جنب مع الـ AI للتحقق من المنطق الطبي.

### 4. الشات بوت الطبي (RAG Chatbot)
- **الهدف**: الإجابة على أسئلة المريض وشرح النتائج.
- **التقنية**: Retrieval-Augmented Generation (RAG).
- **المكونات**:
  - `ChromaDB`: قاعدة بيانات متجهة (Vector DB) لتخزين المعلومات الطبية الموثوقة.
  - `Google Gemini 1.5 Flash`: نموذج لغوي لتوليد الإجابة بأسلوب طبيعي.
- **الميزة**: لا يهلوس (Hallucination) لأنه يعتمد على المصادر المخزنة فقط.

### 5. استخراج البيانات من الصور (OCR)
- **الهدف**: تسهيل الإدخال على المريض بتصوير ورقة التحليل.
- **التقنية**: EasyOCR + OpenCV.
- **العملية**:
  1. معالجة الصورة (Preprocessing) لزيادة التباين.
  2. استخراج النصوص والأرقام.
  3. ربط الأرقام بالتحاليل (Mapping) مثل Creatinine, Urea.

### 6. تصنيف صور الأشعة المقطعية (CT Scan)
- **الهدف**: تمكين الطبيب من تحليل أشعة الكلى المقطعية بسرعة.
- **التقنية**: MobileNetV2 (CNN).
- **المخرجات**: الكشف عن 4 فئات (Normal, Cyst, Stone, Tumor).

### 7. النظام الغذائي الذكي (Smart Diet Planner)
- **الهدف**: تزويد المريض بخطة غذائية آمنة لـ 7 أيام.
- **التقنية**: Google Gemini + KDIGO Nutrition Rules.
- **الميزة**: يراعي بدقة مستويات البوتاسيوم والفسفور الحالية للمريض ويمنع الأطعمة الخطيرة.

##  دورة عمل النظام (Data Flow)

1.  **الطلب (Request)**: يرسل العميل (موبايل/ويب) بيانات المريض وصورة التحليل إلى `api.py`.
2.  **المعالجة (Processing)**:
    - إذا كانت صورة: يقوم `ocr_extractor` بتحويلها لأرقام.
    - إذا كانت أرقام: يتم تنظيفها وتجهيزها في `DataLoader`.
3.  **التنبؤ (Inference)**:
    - `EnsembleModel`: يتوقع وجود المرض وتحديد الاحتمالية.
    - `StagingModel`: يتوقع المرحلة (0-5) بناءً على الأنماط.
    - `GFRCalculator`: يحسب الـ eGFR والمرحلة طبياً.
4.  **الدمج (Aggregation)**: يقوم الـ API بدمج كل النتائج (AI + Clinical).
5.  **التقرير (Reporting)**: يتم إنشاء تقرير PDF مفصل بالحالة والتوصيات.
6.  **الرد (Response)**: يعود الرد للعميل بصيغة JSON نظيفة وجاهزة للعرض.
