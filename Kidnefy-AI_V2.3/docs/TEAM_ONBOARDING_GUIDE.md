# 🚀 Kidnefy-AI: The Ultimate Team Onboarding Guide

أهلاً بك في فريق تطوير **Kidnefy-AI**! 
هذا المستند مصمم خصيصاً ليأخذك من الصفر وحتى الاحتراف في فهم المشروع. إذا كنت لا تعرف أي شيء عن المشروع، فهذا هو المكان الصحيح لتبدأ منه.

---

## 💡 1. ما هو مشروع Kidnefy-AI؟
**الهدف:** بناء "نظام دعم قرار طبي ذكي" (Clinical Decision Support System) لمساعدة الأطباء والمرضى في تشخيص، تصنيف، ومراقبة مرض الكلى المزمن (CKD).

المشروع ليس مجرد "كود يتوقع المرض"، بل هو منصة متكاملة (مستشفى إلكتروني مصغر) تحتوي على 9 محركات ذكاء اصطناعي تعمل معاً لتوفير رعاية طبية كاملة.

---

## 🛠️ 2. التقنيات المستخدمة (Tech Stack)
* **الباك اند (Backend):** Python + FastAPI (سريع جداً ومبني على الـ Asynchronous).
* **تعلم الآلة (Machine Learning):** XGBoost, Scikit-learn (Isolation Forest).
* **التعلم العميق والرؤية الحاسوبية (Deep Learning & CV):** TensorFlow 2.15, MobileNetV2, EasyOCR.
* **الذكاء الاصطناعي التوليدي (GenAI & NLP):** Google Gemini 2.5 Flash, ChromaDB, LangChain.

---

## 🧠 3. الميزات التسعة الأساسية (The 9 Core Features)

هذا ما يفعله النظام بالضبط:

1. **التوقع الطبي (Prediction Ensemble):**
   * يأخذ تحاليل المريض (الكرياتينين، السكر، الضغط، الخ) ويتوقع نسبة الخطر للإصابة بمرض الكلى بدقة **98.5%** باستخدام XGBoost و Deep Learning.
2. **التصنيف السريري (Clinical Staging):**
   * يطبق القواعد الطبية العالمية (KDIGO 2012) ليحسب كفاءة الكلى (eGFR) ويصنف المريض من المرحلة الأولى (G1) للأخيرة (G5).
3. **الاستخراج النصي من الروشتات (OCR):**
   * المريض يصور روشتة التحاليل بالموبايل ويرفعها، والـ AI يقرأ الأرقام من الصورة أوتوماتيكياً عبر `EasyOCR` ويدخلها للنظام.
4. **تحليل الأشعة المقطعية (CT Scan Vision):**
   * موديل `MobileNetV2` مدرب على 12,000 صورة أشعة، يحلل الأشعة المقطعية للكلى ويقول هل هي (طبيعية، فيها حصوة، كيس، أو ورم).
5. **محاكي العلاج (What-If Simulator):**
   * يسمح للطبيب بتجربة خطة علاج وهمية (مثلاً: ماذا لو خفضنا الضغط لـ 120؟) والـ AI يخبره بكم ستقل نسبة الخطر.
6. **مراقبة المرضى والإنذار المبكر (Longitudinal Monitoring):**
   * النظام يحتفظ بتاريخ تحاليل المريض، ويستخدم خوارزمية `Isolation Forest` لاكتشاف التدهور المفاجئ (Fast Progressors) ويرسل إنذاراً للطبيب.
7. **الشات بوت الطبي (Gemini RAG):**
   * شات بوت مبني على `ChromaDB` (قاعدة بيانات موجهة) قرأ كل المراجع الطبية المعتمدة، يجاوب على أسئلة المرضى بدقة تامة دون تأليف (No Hallucination).
8. **مخطط النظام الغذائي الذكي (Smart Diet Planner):**
   * يصمم جدول أكل لمدة 7 أيام للمريض، مع مراعاة القيود الطبية الصارمة (مثلاً: يمنع الأكل الغني بالبوتاسيوم لمرضى المرحلة الرابعة).
9. **إصدار التقارير الطبية (Report Generator):**
   * يطبع تقرير طبي كامل بصيغة HTML/PDF باللغتين العربية والإنجليزية.

## 🏗️ 4. كيف تعمل المعمارية؟ (Architecture Flow)

1. **الفرونت اند (UI):** يرسل طلب (Request) يحتوي على بيانات المريض (مثلاً ملف JSON فيه أرقام التحاليل).
2. **الـ FastAPI (api.py):** يستقبل الطلب، وهو بمثابة "مدير المرور".
3. **التوزيع (Routing):** الـ FastAPI يرسل البيانات للموديل المناسب (مثلاً يرسلها لملف `staging_engine.py` لحساب المرحلة، ثم يرسلها لـ `ml_models.py` لحساب الخطر).
4. **التجميع (Aggregation):** الـ FastAPI يجمع الإجابات من كل الموديلات، ويصيغها في ملف JSON واحد منظم.
5. **الرد (Response):** يعيد الـ JSON إلى الفرونت اند ليعرضه للمستخدم.

## 📁 5. خريطة الملفات (أين تجد الكود؟)

المشروع منظم جداً، لا تلمس الكود قبل أن تعرف أين تذهب:

* `api.py`: **هنا توجد الـ Endpoints!** أي شيء يبدأ بـ `@app.post` أو `@app.get` موجود هنا.
* `config.py`: إعدادات المشروع ومسارات الملفات.
* `.env`: تضع فيه `GEMINI_API_KEY` الخاص بك.
* `src/`: **مجلد الذكاء الاصطناعي.**
   * `src/staging/`: كود القواعد الطبية وحسابات الـ eGFR.
   * `src/models/`: أكواد تشغيل الموديلات (XGBoost).
   * `src/rag/`: أكواد الشات بوت و Gemini.
   * `src/imaging/`: كود تحليل صور الـ CT Scan.
* `models/`: **هنا تحفظ الموديلات بعد تدريبها.** (ملفات `.keras` و `.pkl`). الـ API يقوم بتحميلها من هنا.

## 💻 6. كيف تشغل المشروع على جهازك؟ (Run it Locally)

1. **استنساخ المشروع:**
   ```bash
   git clone https://github.com/amribrahim11vv/Kidnefy-Ai.git
   cd Kidnefy-Ai
   ```
2. **إنشاء البيئة الافتراضية وتثبيت المكتبات:**
   ```bash
   python -m venv .venv
   .venv\Scripts\activate   # (في ويندوز)
   pip install -r requirements.txt
   ```
3. **تجهيز المفتاح السري:**
   أنشئ ملف `.env` واكتب فيه: `GEMINI_API_KEY=your_key_here`
4. **تشغيل السيرفر:**
   ```bash
   uvicorn api:app --reload
   ```
5. **اختبار الأكواد:**
   افتح المتصفح على `http://localhost:8000/docs` وستجد واجهة Swagger جاهزة لتجربة أي ميزة بضغطة زر.
